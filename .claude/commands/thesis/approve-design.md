# 연구설계 승인 (HITL-3)

## 역할

이 커맨드는 **HITL-3 체크포인트**로, 연구설계(Phase 2) 완료 후 사용자가 연구방법론을 검토하고 승인하는 단계입니다.

## 실행 조건

- Phase 2 (Research Design) 완료
- 연구방법론 설계 완료
- `thesis-output/_temp/20-research-design.md` 생성

## 프로세스

### Step 1: 현재 상태 확인

`thesis-output/session.json`을 읽어:

```
- 현재 Phase: phase2 (research_design)
- 연구 유형: [quantitative/qualitative/mixed]
- 이론적 프레임워크: [선택된 이론]
```

### Step 2: 연구설계 결과 제시

#### 2.1 연구 패러다임
- 인식론적 입장
- 연구 접근법

#### 2.2 연구 질문 및 가설
- 최종 연구질문 (RQs)
- 검증 가설 (Hs)

#### 2.3 연구 설계
- 연구 유형
- 데이터 수집 방법
- 분석 방법

#### 2.4 표본 전략
- 모집단 정의
- 표본 크기
- 표집 방법

#### 2.5 측정 도구
- 사용 척도
- 타당도/신뢰도

### Step 3: 사용자 검토 요청

다음 사항에 대해 사용자 확인 요청:

1. **연구 질문 적절성**
   - 연구 목적과 일치하는지
   - 검증 가능한지

2. **방법론 타당성**
   - 연구 질문에 적합한 방법인지
   - 실현 가능한지

3. **윤리적 고려**
   - IRB 승인 필요 여부
   - 인폼드 컨센트 절차

### Step 4: ⭐ 다음 단계 시뮬레이션 모드 선택 (NEW)

연구설계 승인 시, 다음 Phase 3 (논문작성)을 어떻게 시뮬레이션할지 선택:

```markdown
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
📊 다음 단계: Phase 3 (논문작성) 시뮬레이션 모드 선택
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

┌─────────────────────────────────────────────┐
│ 전체 논문(Ch 1-5)을 어떻게 작성할까요?      │
├─────────────────────────────────────────────┤
│                                             │
│ [1] 🚀 Quick Simulation                     │
│     ▪ 출력: 20-30페이지 (Ch 1-5 전체)       │
│     ▪ 시간: 1-2시간                         │
│     ▪ 용도: 전체 흐름 확인, 빠른 검증       │
│     ▪ 품질: 박사급 (Full과 동일) ⭐         │
│     ▪ 수준: 학회 발표 논문 (8-10p)          │
│                                             │
│ [2] 📚 Full Simulation                      │
│     ▪ 출력: 145-155페이지 (Ch 1-5 전체)     │
│     ▪ 시간: 5-7시간                         │
│     ▪ 용도: 최종 학위논문, 학술지 투고      │
│     ▪ 품질: 박사급 (Quick과 동일) ⭐        │
│     ▪ 수준: Journal article (full)          │
│                                             │
│ [3] 🔀 Quick 검토 → Full 완성               │
│     ▪ 1단계: Quick으로 전체 초안 작성       │
│     ▪ 2단계: 초안 검토 및 방향 확인         │
│     ▪ 3단계: Full로 정교화                  │
│     ▪ 시간: 6-9시간                         │
│     ▪ 품질: 최고 (검증 후 완성)             │
│                                             │
└─────────────────────────────────────────────┘

**MANDATORY: 위 3개 시뮬레이션 모드([1] Quick, [2] Full, [3] Both)를 AskUserQuestion의 options에 모두 포함해야 합니다.
어떤 옵션도 생략하지 마십시오.**

⚠️  중요: Quick/Full 모두 품질 기준 동일!
    - Ch 1-5 모두 작성 (완전한 논문)
    - pTCS ≥ 75 (필수)
    - SRCS ≥ 75 (필수)
    - 표절 < 15% (필수)
    - 논리적 완결성 보장
    - 학술적 엄밀성 유지

차이점: 분량과 상세도만 다름
    Quick: 압축된 학회 논문 (20-30p)
    Full: 상세한 학위논문 (150p)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

### Step 5: 결정 사항

```
Phase 3 진행 옵션:
1. ✅ 승인 + Quick Simulation (20-30p)
2. ✅ 승인 + Full Simulation (150p)
3. ✅ 승인 + Both (Quick → Full)

연구설계 수정 옵션:
4. 📝 수정: 방법론 수정 후 재검토
5. 🔄 재설계: 근본적 재설계 필요
```

### Step 6: 승인 처리

승인 시:
1. `session.json` 업데이트:
   - current_phase → phase3
   - simulation_mode → [quick|full|both]
   - writing_mode → [quick|full]
2. 연구설계 확정 기록
3. HITL-3 체크포인트 완료
4. 선택된 시뮬레이션 모드로 Phase 3 시작 안내

## 출력 형식

```markdown
# 연구설계 승인 검토 (HITL-3)

## 1. 연구 질문
### 주 연구질문
RQ: [연구질문]

### 세부 연구질문
1. RQ1: [질문]
2. RQ2: [질문]

## 2. 가설
1. H1: [가설] (근거: [문헌])
2. H2: [가설] (근거: [문헌])

## 3. 연구 방법론
| 항목 | 내용 |
|------|------|
| 연구 유형 | [정량/정성/혼합] |
| 연구 설계 | [횡단/종단/실험 등] |
| 데이터 수집 | [설문/인터뷰/관찰 등] |
| 표본 크기 | [N명/개] |
| 분석 방법 | [SEM/회귀/주제분석 등] |

## 4. 측정 도구
| 변수 | 척도 | 출처 | 신뢰도 |
|------|------|------|--------|

## 5. 승인 결정
[ ] 승인하고 Phase 3 진행
[ ] 수정 요청
[ ] 재설계 필요
```
