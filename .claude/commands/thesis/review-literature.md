# 문헌검토 승인 (HITL-2)

## 역할

이 커맨드는 **HITL-2 체크포인트**로, 문헌검토(Phase 1) 완료 후 사용자가 결과를 검토하고 승인하는 단계입니다.

## 실행 조건

- Phase 1 (Literature Review) Wave 1-5 완료
- 품질 등급 C 이상 (SRCS 75점+)
- research-synthesis.md 생성 완료

## 프로세스

### Step 1: 현재 상태 확인

`thesis-output/session.json`을 읽어 현재 워크플로우 상태를 확인합니다:

```
- 현재 Phase: phase1 (literature_review)
- 완료된 Wave: Wave 1-5
- 품질 등급: [A/B/C/D/F]
```

### Step 2: 문헌검토 결과 요약

사용자에게 다음 정보를 제시합니다:

#### 2.1 품질 보고서 요약
`thesis-output/_temp/quality-report.md`에서:
- SRCS 종합 점수
- 품질 등급
- 임계값 미달 클레임 수

#### 2.2 문헌검토 핵심 발견
`thesis-output/research-synthesis.md`에서:
- 핵심 이론적 발견 (3-5개)
- 핵심 실증적 발견 (3-5개)
- 주요 연구 갭 (3-5개)

#### 2.3 개념적 모델
`thesis-output/_temp/14-conceptual-model.md`에서:
- 연구 변수
- 가설

### Step 3: 사용자 검토 요청

다음 사항에 대해 사용자 확인을 요청합니다:

1. **문헌검토 범위 적정성**
   - 핵심 문헌이 누락되지 않았는지
   - 검토 범위가 연구 주제에 적합한지

2. **연구 방향 적합성**
   - 제안된 연구 질문이 적절한지
   - 가설이 문헌에 기반하고 있는지

3. **품질 수준 승인**
   - SRCS 점수가 허용 가능한지
   - 추가 보완이 필요한지

### Step 3-1: ⚠️ Mode F 프로포절 모순 확인 [Mode F Only]

session.json의 mode가 "proposal"인 경우, 문헌검토 결과와 프로포절 계획 간
모순 여부를 확인하고 사용자에게 알립니다.

**Flag-and-Follow 정책**:

문헌검토가 완료된 후, 프로포절에서 추출된 연구 계획과 문헌검토 결과 간
정합성을 자동 검증합니다.

**확인 사항**:
1. 프로포절의 연구질문이 문헌에 의해 지지되는지
2. 프로포절의 방법론이 선행연구 패턴과 정합하는지
3. 프로포절의 이론적 프레임워크가 최신 문헌과 부합하는지

**모순 발견 시**:
```
⚠️  Flag-and-Follow: 프로포절과 문헌검토 간 [N]개 모순 발견

1. [모순 항목]: [프로포절 내용] vs [문헌검토 결과]
   → 영향도: [높음/중간/낮음]

선택지:
  (1) 프로포절 유지 - 근거를 직접 제시해주세요
  (2) 문헌검토 결과 반영 수정
  (3) 절충안 도출
```

**모순 미발견 시**: 별도 안내 없이 Step 4로 자동 진행합니다.

### Step 5: 결정 사항

사용자에게 선택지 제시:

```
Phase 2 진행 옵션:
1. ✅ 승인 + Quick Simulation
2. ✅ 승인 + Full Simulation
3. ✅ 승인 + Both (Quick → Full)

문헌검토 수정 옵션:
4. 📝 수정 요청: 특정 부분 보완 후 재검토
5. ➕ 문헌 추가: 추가 문헌 검토 후 재검토
6. ❌ 재시작: 문헌검토 처음부터 재시작
```

### Step 6: 승인 처리

승인 시:
1. `session.json` 업데이트:
   - current_phase → phase2
   - simulation_mode → [quick|full|both]
2. HITL-2 체크포인트 완료 기록
3. 선택된 시뮬레이션 모드로 Phase 2 시작 안내
