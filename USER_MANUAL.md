# 1. 시작하기

## 1.1 이 시스템은 무엇인가요?

### 한 문장 요약
**당신의 연구 아이디어를 입력하면, 그 아이디어로 실제 논문을 완성했을 때의 결과를 박사급 수준의 글로 미리 써서 보여주는 연구 시뮬레이터입니다.**

---

### 📖 시뮬레이션의 핵심: "직접 읽고 확인"

```
전통적 연구 과정:
┌──────────────────────────────────────────┐
│  아이디어 → 6개월 작업 → 논문 완성      │
│             ↓                            │
│         "음... 이게 아닌 것 같은데..." │
│             ↓                            │
│         너무 늦게 발견! 😢              │
└──────────────────────────────────────────┘

시뮬레이션 방식:
┌──────────────────────────────────────────┐
│  아이디어 → 1일 시뮬레이션               │
│             ↓                            │
│  150페이지 논문 초안을 직접 읽어봄      │
│             ↓                            │
│  "오, 이 부분은 좋은데 저 부분은..."   │
│  "3장을 이렇게 수정하면 더 좋겠다"     │
│             ↓                            │
│  수정 요청 → 재시뮬레이션 (1-2시간)    │
│             ↓                            │
│  수정된 논문을 다시 읽어봄              │
│             ↓                            │
│  만족할 때까지 반복! ✅                 │
└──────────────────────────────────────────┘
```

**핵심**: 점수나 숫자가 아니라, **실제 논문을 미리 읽어보고** 판단할 수 있습니다.

---

### 💡 "시뮬레이션"의 진짜 의미

**❌ 오해**: "AI가 점수만 계산해준다"
**✅ 실제**: "AI가 실제 논문을 완성해서 보여준다"

```
예시: "AI (Artificial Intelligence, 인공지능)가 조직혁신에 미치는 영향" 연구

일반적 기대:
  입력: "AI와 혁신"
  출력: "실현가능성 점수 82점"
  → 이게 뭘 의미하는지 모호함

실제 시뮬레이션:
  입력: "AI가 조직혁신에 미치는 영향"
  출력:
    📄 150페이지 논문 초안 (Word 파일)
    ├─ Chapter 1: Introduction (15페이지)
    │   - 연구 배경: "AI 기술의 발전과 함께..."
    │   - 연구 필요성: "기존 연구는 주로..."
    │   - 연구 질문: "본 연구는 다음을 탐구한다..."
    │
    ├─ Chapter 2: Literature Review (40페이지)
    │   - 2.1 AI Adoption (12페이지)
    │   - 2.2 Innovation Theories (15페이지)
    │   - 2.3 Research Gap (8페이지)
    │   - ... 실제 문단들을 읽어볼 수 있음
    │
    ├─ Chapter 3: Methodology (20페이지)
    │   - 3.1 Research Design: "본 연구는 횡단연구..."
    │   - 3.2 Sample: "제조업 SME (Small and Medium Enterprise, 중소기업) 285개 기업..."
    │   - ... 구체적인 설계를 확인
    │
    ├─ Chapter 4: Results (45페이지)
    │   - 가상 데이터로 예상 결과 시뮬레이션
    │
    └─ Chapter 5: Conclusion (15페이지)

  → 실제로 읽어보고 판단:
    "오, 2.3장의 연구 갭 분석이 좋네"
    "근데 3.2장 표본 설계가 너무 복잡한데?"
    "4장 결과 부분을 더 간결하게 만들어야겠다"
```

**이것이 시뮬레이션입니다**: 숫자가 아니라 **실제 글을 읽고 확인**합니다.

### 🎯 3가지 핵심 기능

```
┌─────────────────────────────────────────────────┐
│  1️⃣  연구질문 시뮬레이션                        │
│     "이 주제로 연구하면 어떤 질문이 가능할까?"  │
│                                                 │
│     입력: 연구 주제                             │
│     출력: 3-5개의 구체적 연구질문               │
│           각 질문마다 2-3페이지 설명            │
│           - 왜 이 질문인가?                     │
│           - 어떤 문헌이 뒷받침하는가?           │
│           - 어떤 방법론이 적합한가?             │
│     → 읽어보고 가장 좋은 질문 선택             │
│                                                 │
├─────────────────────────────────────────────────┤
│  2️⃣  문헌검토 시뮬레이션                        │
│     "이 질문으로 논문을 쓰면 어떤 문헌검토가    │
│      필요할까?"                                 │
│                                                 │
│     입력: 연구질문                              │
│     출력: 40-50페이지 문헌검토 초안             │
│           - 핵심 이론 설명 (15-20페이지)        │
│           - 실증 연구 종합 (15-20페이지)        │
│           - 연구 갭 분석 (8-10페이지)           │
│           - 개념적 모델 (5페이지)               │
│     → 실제 문단을 읽어보고 판단                │
│       "이 이론이 적합한가?"                     │
│       "이 갭이 흥미로운가?"                     │
│                                                 │
├─────────────────────────────────────────────────┤
│  3️⃣  전체 논문 시뮬레이션                       │
│     "이 설계로 논문을 완성하면 어떻게 될까?"    │
│                                                 │
│     입력: 연구설계                              │
│     출력: 150페이지 논문 전체 초안              │
│           - 5개 장 완전 작성                    │
│           - 참고문헌 100-150개                  │
│           - 그림/표 20-30개                     │
│     → Word 파일로 다운로드하여                 │
│       처음부터 끝까지 읽어보기                  │
│       마치 완성된 논문처럼                      │
└─────────────────────────────────────────────────┘
```

### 👨‍🔬 당신의 역할 vs AI의 역할

| 당신 (연구자) | AI (시뮬레이터) |
|---------------|-----------------|
| 🧠 아이디어 제공 | 📝 그 아이디어로 실제 논문 작성 |
| 📖 **시뮬레이션 결과 읽기** | 📊 박사급 수준의 글로 제시 |
| ✍️  수정 요청 | 🔄 수정된 버전 재작성 |
| 🎯 최종 판단 (만족/불만족) | 💡 만족할 때까지 재시뮬레이션 |
| 📚 채택할 문헌 선택 | 🔍 선택된 문헌 기반 재구성 |
| 🔬 연구설계 결정 | 📐 결정된 설계로 전체 작성 |

**핵심 원칙**:
```
당신이 읽고 판단하고, AI는 써서 보여줍니다.
당신이 "이렇게 수정해"라고 하면, AI는 수정해서 다시 보여줍니다.
점수는 참고용일 뿐, 진짜 판단은 '실제 글을 읽고' 합니다.
```

### 누가 사용하면 좋은가요?

✅ **이런 분들에게 최적**:
- "이 주제로 논문을 쓰면 어떻게 될까?" 궁금한 분
- 실제로 써보기 전에 미리 확인하고 싶은 분
- 여러 방향을 빠르게 시도해보고 싶은 분
- 논문 초안을 읽고 피드백하는 것을 선호하는 분

❌ **이런 분들에게는 비추천**:
- 긴 글을 읽기 싫은 분 (시뮬레이션 결과는 수십 페이지)
- AI 결과를 그대로 제출하려는 분 (표절!)
- 시뮬레이션 결과를 검토할 시간이 없는 분

## 1.3 첫 실행 예시

### 시나리오: "AI가 조직 혁신에 미치는 영향" 연구하기

**1. 시작**
```bash
> 시작하자

System: 연구 주제를 입력해주세요.

> Impact of AI on organizational innovation
```

**2. 첫 시뮬레이션 (30분 대기)**
```
System: 🔄 시뮬레이션 중...
        - 관련 논문 127편 분석 중...
        - 연구 동향 파악 중...
        - 연구질문 도출 중...
```

**3. 결과 확인 - 실제 읽기!**
```
System: ✅ 시뮬레이션 완료

        📄 00-topic-exploration.md (8페이지) 생성
        → 이 파일을 직접 열어서 읽어보세요!

        ───────────────────────────────────────
        파일 내용 미리보기:

        # Research Topic Exploration

        ## Topic Analysis
        The intersection of artificial intelligence (AI)
        and organizational innovation represents a rapidly
        evolving research domain. Recent studies (Smith et
        al., 2023; Johnson & Lee, 2024) highlight...

        ## Proposed Research Questions

        ### RQ1 (Research Question 1, 연구질문 1): AI Adoption and Innovation Processes
        **Question**: How does AI adoption influence
        organizational innovation processes?

        **Rationale**: While existing literature documents
        a positive relationship between AI and innovation
        outcomes (Chen, 2023; Park et al., 2024), the
        underlying mechanisms remain underexplored...

        **Literature Support**:
        - 15 studies confirm positive impact (r=.42-.68)
        - Gap: Process-level analysis lacking
        - Opportunity: Mediating factors unexplored

        **Suggested Methodology**: Longitudinal case study
        or survey-based quantitative research...

        [계속 2-3페이지...]

        ### RQ2: Mediating Factors... [3페이지]
        ### RQ3: Moderating Variables... [3페이지]
        ───────────────────────────────────────

        → thesis-output/impact-of-ai.../00-topic-exploration.md
          파일을 열어서 전체를 읽어보세요!
```

**4. 실제 파일 읽기 (10분)**
```
당신의 판단 (파일을 읽은 후):

  ✅ RQ1: 흥미롭지만 선행연구가 많음 (경쟁 심함)
  ✅ RQ2: 매개요인 분석 - 이게 가장 신선함!
  ✅ RQ3: 조절변수 - 너무 복잡할 것 같음

  결정: "RQ2로 가되, 중소기업에 초점 맞추자"
```

**5. 수정 요청**
```
You: RQ2를 다음과 같이 수정해서 다시 작성해주세요:
     - 중소기업(SME)에 초점
     - 혁신 성과를 재무 성과로 구체화
     - 제조업으로 한정

     전체를 다시 시뮬레이션해주세요.
```

**6. 재시뮬레이션 결과 (10분 후)**
```
System: ✅ 재시뮬레이션 완료

        📄 00-topic-exploration-revised.md (8페이지)

        주요 변경사항:
        ├─ RQ2 수정: SME + 재무성과 + 제조업
        ├─ 관련 문헌 23편 추가 분석
        ├─ 방법론 제안: 설문조사 285개 기업
        └─ 예상 기여: SME 특화 연구로 차별화

        → 파일을 다시 읽어보세요!
```

**7. 수정된 파일 읽기 후 판단**
```
당신의 판단:

  📖 읽어보니...
  ✅ SME 초점 → 타겟이 명확해짐
  ✅ 재무성과 → 측정 가능성 향상
  ✅ 제조업 → 실증 가능성 증가

  만족! "이걸로 가자"
```

**8. 다음 단계 진행**
```
You: y (승인)

System: ✅ 연구질문 확정
        다음: Phase 1 문헌검토 시뮬레이션

        이 질문으로 논문을 쓴다면 어떤 문헌검토가
        필요할지 40-50페이지로 작성합니다.

        예상 시간: 4-6시간
        시작할까요?
```

---

**핵심 포인트**:
- ✅ 점수가 아니라 **실제 8페이지 문서를 읽었습니다**
- ✅ 읽고 판단했습니다: "RQ2가 좋다"
- ✅ 수정 요청 후 **수정된 8페이지를 다시 읽었습니다**
- ✅ 만족할 때까지 반복 가능

이것이 **"시뮬레이션"**입니다!

## 1.4 시뮬레이션 모드 상세 가이드 ⭐

### 시뮬레이션이란 정확히 무엇인가?

```
┌─────────────────────────────────────────────────┐
│  시뮬레이션 = 실제 논문을 미리 써보기           │
├─────────────────────────────────────────────────┤
│  입력: 당신의 아이디어 (주제/질문/설계...)     │
│    ↓                                            │
│  AI가 실제로 논문을 작성                        │
│    ↓                                            │
│  출력: 박사급 수준의 완성된 글                  │
│        (수십-수백 페이지)                       │
│    ↓                                            │
│  당신이 읽고 판단:                              │
│  "이 부분은 좋은데..."                         │
│  "저 부분은 수정이 필요해..."                  │
│    ↓                                            │
│  수정 요청 → 재시뮬레이션 → 다시 읽기          │
└─────────────────────────────────────────────────┘
```

**점수는 참고용일 뿐**:
- pTCS (predicted Thesis Confidence Score, 예측 논문 신뢰도 점수), SRCS (Source-Reliability-Confidence-Scope, 출처-신뢰도-확신-범위 평가) 같은 점수도 제공됩니다
- 하지만 진짜 판단은 **실제 글을 읽고** 합니다
- "82점"보다 "Chapter 2의 이론 부분이 부족함"이 더 유용합니다

### 🤖 Smart Mode (자동 모드 선택) (NEW)

```
│  🎯 최고 품질 (시간 충분)                       │
│  → Quick (탐색) → AlphaGo (비교) → Full (완성) │
│     이유: 단계적 정교화                         │
│                                                 │
└─────────────────────────────────────────────────┘
```

**실전 워크플로우 예시**:

```bash
# Scenario 1: 빠른 전체 완성 (5-7시간)
당신: "전체를 Quick으로 완성해줘"
결과: 20-30페이지 압축 논문 (pTCS 75+)
     → 필요하면 나중에 Full로 확장

# Scenario 2: 최고 품질 (15-20시간)
1. Quick으로 3가지 방법론 탐색 (3h)
2. AlphaGo로 최적안 선택 (4h)
3. Full로 상세 완성 (7h)
4. 검토 후 수정 (2h)
결과: 150페이지 완전한 논문 (pTCS 85+)

# Scenario 3: Autopilot (14-15시간)
당신: "Autopilot으로 자동 실행"
AI: [자동으로 Quick/Both/Full 선택하여 진행]
결과: 최적 경로로 완성 (사용자 개입 0회)
```

## 2.2 7가지 시작 모드 = 시뮬레이션 시작점

각 모드는 **어디서부터 시뮬레이션할지**를 결정합니다.

### 🎯 Mode 1: Topic

**시뮬레이션 시작점**: Phase 0 (처음부터)

**AI가 작성하는 것**:
```
Step 1: 📄 8페이지 - 연구질문 탐색
        → 읽고 선택

Step 2: 📄 162페이지 - 문헌검토 전체
        → 핵심 45페이지 읽고 승인

Step 3: 📄 25페이지 - 연구설계
        → 읽고 승인

Step 4: 📄 150페이지 - 전체 논문
        → 읽고 수정 요청

총: 약 345페이지 작성
당신: 약 230페이지 읽기
```

---

### ❓ Mode 2: Question

**시뮬레이션 시작점**: Phase 1 (연구질문 스킵)

**AI가 작성하는 것**:
```
Step 1: 📄 162페이지 - 문헌검토 전체
Step 2: 📄 25페이지 - 연구설계
Step 3: 📄 150페이지 - 전체 논문

총: 약 337페이지 작성
당신: 약 220페이지 읽기
```

---

### 📚 Mode 3: Review

**시뮬레이션 시작점**: Phase 2 (문헌검토 스킵)

**AI가 작성하는 것**:
```
Step 1: 📄 25페이지 - 연구설계
Step 2: 📄 150페이지 - 전체 논문

총: 약 175페이지 작성
당신: 약 175페이지 읽기
```

**필요한 것**: 당신이 작성한 문헌검토 (30-50페이지)를 `user-resource/` 폴더에 저장

---

### 🎓 Mode 4: Learning

**시뮬레이션 시작점**: Phase 0 (샘플 데이터)

**AI가 작성하는 것**:
```
샘플 주제: "Digital transformation and innovation"

📄 전체 345페이지 (샘플)
  - 각 단계마다 자세한 설명 추가
  - "이 부분은 이런 이유로 작성되었습니다"
  - "실제 사용 시에는 이렇게 수정하세요"
```

**장점**: 비용 없이 전체 프로세스 체험, 실제 결과물 형태 확인

---

### 📄 Mode 5: Paper-Upload ⭐

**시뮬레이션 시작점**: 논문 분석 → 새로운 연구 시뮬레이션

**AI가 작성하는 것**:
```
Step 1: 📄 20페이지 - 업로드된 논문 심층 분석
        - 연구 맥락, 이론, 방법론, 결과, 한계
        → 읽고 확인

Step 2: 📄 10페이지 - 연구 갭 식별
        - 4-5개 확장 가능 방향 제시
        → 읽고 선택

Step 3: 📄 15페이지 - 새로운 가설 도출
        - 선택한 방향으로 가설 3-5개
        → 읽고 확정

Step 4: 📄 25페이지 - 새로운 연구설계
        → 읽고 승인

Step 5: 📄 150페이지 - 새로운 논문 전체
        → 읽고 수정

총: 약 220페이지 작성
당신: 약 220페이지 읽기
```

**사용 예시**:
```bash
/thesis:start paper-upload --paper-path user-resource/smith-2023.pdf

→ Smith(2023) 논문을 분석하여
  "이 연구를 중소기업으로 확장하면?"
  "이 연구를 질적 방법으로 재설계하면?"
  등을 시뮬레이션
```

---

### 📋 Mode 6: Proposal-Upload

**시뮬레이션 시작점**: 프로포절 분석 → 추출된 계획으로 시뮬레이션

**AI가 작성하는 것**:
```
Step 1: 📄 업로드된 프로포절 파싱
        - 연구질문, 가설, 방법론, 변수 자동 추출
        - 완성도 평가 (어디까지 준비되었는지)
        → 읽고 확인

Step 2: 추출된 계획을 기반으로
        준비된 단계부터 시뮬레이션 시작
        (문헌검토 완료 → Phase 2부터,
         설계까지 완료 → Phase 3부터)
```

**언제 사용**: 이미 연구 계획서(프로포절)가 있을 때
```bash
/thesis:start-proposal-upload --proposal-path user-resource/my-proposal.pdf
```

---

### 💬 Mode 7: Custom-Input (자유 입력)

**시뮬레이션 시작점**: 자유 텍스트에서 연구 요소 추출 → 적절한 모드로 자동 연결

**AI가 작성하는 것**:
```
Step 1: 당신의 자유 입력 분석
        - 연구 주제, 질문, 방법론 선호 등을 자동 추출
        - 가장 적합한 모드(Topic/Question/Review 등)로 연결
        → 읽고 확인

Step 2: 연결된 모드의 워크플로우로 진행
```

**언제 사용**: 형식에 맞추기 어렵거나, 자연스럽게 설명하고 싶을 때
```
예시 입력:
  "저는 제조업 SME에서 AI 도입이 혁신에 미치는 영향을
   연구하고 싶습니다. 양적연구를 선호하고,
   동적역량 이론을 활용하고 싶어요."

→ 시스템이 주제, 방법론 선호, 이론을 추출하여
  적절한 모드로 자동 연결
```

```
# Literature Review Synthesis

## Executive Summary

This comprehensive review analyzed 127 articles examining
the relationship between artificial intelligence (AI)
adoption and organizational innovation, with specific focus
on small and medium enterprises (SMEs) in manufacturing.

Key findings:
1. Strong positive relationship (meta-analytic r=.52, 95% CI (Confidence Interval, 신뢰구간)
   [.48, .56]) between AI adoption and innovation outcomes
2. Three theoretical perspectives dominate: Innovation
   Diffusion Theory, Resource-Based View, and Dynamic
   Capabilities Framework
3. Four critical research gaps identified, with mediating
   mechanisms representing the most significant opportunity
4. Methodological limitations: 87% cross-sectional, only
   13% longitudinal designs
```

```
6. 여러 번 수정
7. 표절 검사
8. 지도교수 검토
9. 최종 제출
```

## 4.2 기술적 질문

### Q5: 파일이 너무 많아요. 어떤 걸 읽어야 하나요?

**A: ⭐ 표시된 핵심 파일만 읽으면 됩니다.**

```
📁 thesis-output/your-topic-2026-01-29/

필수 읽기: ⭐⭐⭐
  📄 research-synthesis.md (45페이지)
     → 문헌검토 전체 종합

  📄 research-design-report.md (25페이지)
     → 연구설계 확정본

  📄 dissertation-full-en.docx (150페이지)
     → 전체 논문 통합본

선택 읽기: ⭐
  📄 00-topic-exploration.md (8페이지)
     → 연구질문 탐색 (이미 확정됨)

  📄 Wave 1-5 개별 파일 (각 8-12페이지)
     → 문헌검토 상세 (궁금하면 읽기)

참고용:
  📄 session.json
     → 진행 상태 (사람이 읽는 용도 아님)

  📄 quality-report.md
     → 품질 점수 (참고만)
```

### Q6: 영어를 못하는데 괜찮나요?

**A: 한국어 번역본이 자동 생성됩니다.**

```
모든 파일이 2개 버전으로:
  📄 research-synthesis.md (영어)
  📄 research-synthesis-ko.md (한국어) ⭐

  📄 dissertation-full-en.docx (영어)
  📄 dissertation-full-ko.docx (한국어) ⭐

한국어 버전 읽기:
  ✅ 전문 번역 품질
  ✅ 학술 용어 정확
  ✅ 인용/DOI는 영어 유지

권장:
  1. 한국어 버전으로 읽고 이해
  2. 영어 버전으로 정확성 확인
  3. 수정 요청은 한국어로 해도 됨
```

### Q7: 시뮬레이션이 너무 오래 걸려요

**A: 정상입니다. 품질을 위해 시간이 필요합니다.**

```
Phase별 시간:
  Phase 0: 30분 (연구질문 3개 작성)
  Phase 1: 4-6시간 (문헌검토 162페이지 작성)
  Phase 2: 2-3시간 (연구설계 25페이지 작성)
  Phase 3: 3-5시간 (전체 논문 150페이지 작성)
  Phase 4: 1-2시간 (투고전략 15페이지 작성)

왜 이렇게 오래?
  ✅ 박사급 수준의 글 작성
  ✅ 100+ 논문 분석
  ✅ 품질 검증 (pTCS, SRCS)
  ✅ 재시뮬레이션 (품질 미달 시)

빠르게 하려면?
  옵션 1: Review 모드
    → Phase 1 스킵 (4-6시간 절약)

  옵션 2: Learning 모드
    → 샘플로 빠른 체험

  옵션 3: 단계별 실행
    → 필요한 Phase만 실행
```

## 4.3 활용 관련 질문

### Q8: 문헌을 직접 선택하고 싶어요

**A: Level 3 시뮬레이션 모드를 사용하세요!**

```
Level 3 사용법:

1. 문헌 검색 결과 받기
   System: 📄 127편 목록 (12페이지)

2. 목록 읽기 (30분-1시간)
   당신: 각 논문의 초록 확인

3. 선택
   옵션 A: 추천 23편 승인
     > "추천 논문들로 진행"

   옵션 B: 직접 선택
     > "논문 #1, #5, #8, #12... 이 15편만"

   옵션 C: 추가
     > "추천 23편 + 내가 읽은 Smith(2023)"
     > user-resource/smith-2023.pdf 업로드

4. 선택된 문헌으로 시뮬레이션
   → 당신이 선택한 문헌만 사용!
```

# 5. 고급 활용

## 5.1 시뮬레이션 레벨 전환

시작 후에도 레벨 변경 가능:

```
초기: Level 1 (전체 자동)으로 시작
  ↓
중간에 Level 3으로 전환:
  > 문헌 선택을 직접 하고 싶습니다.
    현재까지 발견된 127편 목록을 보여주세요.

  → Level 3 모드로 전환
  → 문헌 선택 → 이후 계속
```

## 5.2 반복 시뮬레이션 전략

**상황**: 방향을 확신할 수 없을 때

**전략**: 여러 옵션을 동시에 시뮬레이션

```
방법 1: RQ별 시뮬레이션
  > RQ1, RQ2, RQ3를 각각 별도로
    문헌검토까지 시뮬레이션해주세요.

    3개 세션 생성:
    - Session A: RQ1 기반
    - Session B: RQ2 기반
    - Session C: RQ3 기반

  → 각 45페이지 문헌검토 읽기
  → 가장 좋은 것 선택
  → 선택한 것만 논문까지 진행

방법 2: 설계별 시뮬레이션
  > 양적, 질적, 혼합 3가지를
    각각 Chapter 3까지 작성해주세요.

  → 3가지 방법론 읽고 비교
  → 가장 현실적인 것 선택
```

---

## 5.3 점진적 개선 전략

**목표**: 최고 품질 달성

```
Round 1: 빠른 시뮬레이션
  - Level 1 (전체 자동)
  - 150페이지 읽기
  - 큰 그림 파악
  - 점수: 78점

Round 2: 핵심 개선
  - 가장 약한 Chapter 3개 재시뮬레이션
  - 각 Chapter 정독
  - 점수: 83점

Round 3: 세부 조정
  - 각 Chapter의 약한 섹션만 재시뮬레이션
  - 섹션별 정밀 검토
  - 점수: 87점

Round 4: 최종 다듬기
  - 표현 수정
  - 당신의 아이디어 추가
  - 최종 점수: 90점+
```

## 5.4 커맨드 레퍼런스

### 시작하기

```bash
/thesis:start topic "..."              # 주제로 시작
/thesis:start question "..."           # 연구질문으로 시작
/thesis:start paper-upload             # 논문 업로드로 시작
/thesis:start-proposal-upload          # 프로포절 업로드로 시작
/thesis:quick-start                    # 자연어로 빠른 시작
/thesis:resume                         # 이전 세션 이어서 진행
```

### 상태 확인

```bash
/thesis:status                         # 현재 진행 상태
/thesis:progress                       # 전체 진행률 시각화
```

### 각 Phase 실행

```bash
/thesis:run-literature-review          # Phase 1: 문헌검토 시뮬레이션
/thesis:run-research-design            # Phase 2: 연구설계 시뮬레이션
/thesis:run-writing                    # Phase 3: 논문작성 시뮬레이션
/thesis:run-publication                # Phase 4: 투고전략 시뮬레이션
/thesis:autopilot                      # 전체 자동 실행 (HITL 자동승인, 명시적 명령 필요)
```

### 읽고 판단하기 (시뮬레이션 포인트)

```bash
/thesis:approve-topic                  # 연구질문 승인 (포인트 #1)
/thesis:review-literature              # 문헌검토 승인 (포인트 #2)
/thesis:set-research-type              # 연구유형 선택 (포인트 #3)
/thesis:approve-design                 # 연구설계 승인 (포인트 #4)
/thesis:approve-outline                # 논문 아웃라인 승인 (포인트 #5-6)
/thesis:review-chapter [N]             # 챕터별 검토/수정 요청 (포인트 #7)
/thesis:finalize                       # 최종 완료 (포인트 #8)
```

### 파일 생성 및 변환

```bash
/thesis:export-docx                    # Word 파일 내보내기
/thesis:translate                      # 한국어 번역 생성
```

### 품질 확인 (참고용)

```bash
/thesis:validate-all                   # 전체 품질 검증
/thesis:check-plagiarism               # 표절 검사
```

# 6. 용어 사전

| 용어 | 의미 |
|------|------|
| **시뮬레이션** | 실제 논문을 미리 작성해보는 것 |
| **시뮬레이션 포인트** | AI 작성 결과를 읽고 판단하는 지점 |
| **레벨** | 시뮬레이션 수준 (1: 전체 자동 ~ 5: 단계별 조정) |
| **재시뮬레이션** | 수정 요청 후 다시 작성 |
| **핵심 파일** | ⭐ 표시된 필수 읽기 파일 |
| **AI (Artificial Intelligence, 인공지능)** | 논문 작성을 시뮬레이션하는 핵심 기술 |
| **APA (American Psychological Association, 미국심리학회)** | 사회과학 분야의 대표적 인용 형식 |
| **CI (Confidence Interval, 신뢰구간)** | 통계적 추정의 신뢰 범위 |
| **CLI (Command Line Interface, 명령줄 인터페이스)** | 텍스트 기반 명령 입력 방식 |
| **DOI (Digital Object Identifier, 디지털 객체 식별자)** | 학술 논문의 고유 식별 코드 |
| **IDT (Innovation Diffusion Theory, 혁신 확산 이론)** | 혁신이 조직 내/간에 확산되는 과정을 설명하는 이론 |
| **MLA (Modern Language Association, 현대언어학회)** | 인문학 분야의 대표적 인용 형식 |
| **PLS-SEM (Partial Least Squares Structural Equation Modeling, 편최소제곱 구조방정식 모델링)** | 변수 간 복잡한 관계를 분석하는 통계 기법 |
| **pTCS (predicted Thesis Confidence Score, 예측 논문 신뢰도 점수)** | 논문 품질을 0-100 점수로 평가하는 지표 (참고용, 읽기가 우선) |
| **RQ (Research Question, 연구질문)** | 논문에서 탐구하고자 하는 핵심 질문 |
| **SME (Small and Medium Enterprise, 중소기업)** | 중소 규모의 기업 |
| **SRCS (Source-Reliability-Confidence-Scope, 출처-신뢰도-확신-범위 평가)** | 연구 주장의 품질을 4개 축으로 평가하는 지표 (참고용, 읽기가 우선) |
| **Win Rate (승률, 논문 통과 가능성)** | AlphaGo 평가에서 특정 연구 설계의 심사 통과 예측 확률 |
